# mvp_designer

Purpose: Turn an approved opportunity into a concrete MVP plan.

## When To Use
- A direction is selected and you need to define what to build.

## Inputs
- Selected opportunity from `opportunity_mapper`.
- Datasets shortlist, team skills, timebox.
- Build guidance in `04_build_guides/**`.

## Outputs
- User flow (text description) and primary success path.
- Architecture diagram (text), component list, and data flows.
- Scoped feature list with acceptance criteria and out-of-scope items.
- 1–2 stretch goals and a demo‑ready checkpoint plan.

## Process
1) Choose a simple, reliable architecture from build guides.
2) Define the core user journey and constrain scope to demo essentials.
3) Map datasets/APIs and note stubs/mocks where needed.
4) Enforce A Thriving City Hall scope limits: no RVA311 API; no official City submission; advisory-only procurement.
5) Produce acceptance criteria and a day‑by‑day build plan.
