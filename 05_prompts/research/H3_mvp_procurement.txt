MVP Feasibility H3 — Feasible MVP for Procurement Contract Review

Define the feasible minimum viable prototype for the procurement risk review problem statement, given all known constraints.

CONSTRAINTS RECAP:
- City Contracts Socrata CSV available (all 9 columns via direct download) — start date, end date, vendor, department, amount fields exist
- SODA API has a known column bug — use CSV download
- SAM.gov requires free API key (register before hackathon)
- VITA contracts are web-only — no API
- No pre-staged sample contract PDFs (PDF extraction has a hard dependency on this)
- No named City procurement champion (this is a demo credibility challenge)
- Tool must be advisory only — no compliance determination

THREE APPROACHES TO EVALUATE:

A) Contract Expiry Dashboard (lowest dependency):
- Ingest the City Contracts CSV
- Normalize dates; calculate days-to-expiry
- Build sortable/filterable table with 30/60/90 day filters
- No external API required
- Time estimate: 8–12 hours
- Demo: "Load the data → filter to next 60 days → see 8 contracts → click one for details"

B) PDF Contract Extractor (highest dependency):
- Upload PDF → extract expiration date, renewal window, pricing terms using LLM
- CRITICAL DEPENDENCY: requires sample PDFs before/during hackathon
- Time estimate: 10–14 hours (if PDFs are available from day 1)
- Demo: "Before: scroll through 40 pages. After: extracted summary in 10 seconds."

C) Cross-Source Finder (medium dependency):
- City Contracts CSV + SAM.gov API (requires key) + eVA CSV
- Time estimate: 14–18 hours
- Demo: "Search 'janitorial services' → see City, state, and federal contracts side by side"

Output
- Feasibility assessment for each approach
- Recommended approach with minimum viable feature set
- Critical dependencies checklist (must-haves before building starts)
