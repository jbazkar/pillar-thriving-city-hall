Prior Art E1 — National 311 Open Data Tools, Chatbots, and Interfaces

Investigate national prior art for 311 data tools, service navigation chatbots, and plain-language request interfaces. Focus on tools that address the same problem as Richmond's resident service navigation statement: helping residents identify the right category and department before submitting a request.

Key examples to investigate:
- NYC 311 (data.cityofnewyork.us) — open API, extensive data, any AI-assisted features?
- Boston 311 / SeeClickFix — what features does the SeeClickFix platform offer for category guidance?
- San Francisco SF311 — Open311 API, any smart routing features?
- Chicago Open311 — what does the Open311 standard specify for service type documentation?
- Code for America projects related to 311 data and service navigation
- Any civic hackathon projects that have built 311 category helpers or chatbots

For each tool, document:
- What problem it solves
- How it handles category selection or plain-language input
- What data it uses
- Whether it is still active
- Key lessons (what worked, what failed)

Also investigate: what does the Open311 standard specify for service type description fields — could those field definitions be adapted for a Richmond 311 helper without live data access?

Output
- Inventory of prior art tools with status and key lessons
- Most applicable models for a Richmond hackathon prototype
- Failure modes to avoid
